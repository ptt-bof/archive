話者 : (1) 川崎 修
題目 : (1) 帯域利用状況に対応した経路制御に関する考察
概要： (1) 
企業やキャンパスネットワークで一般的に使用されている動的経路制御に
OSPF(Open Shortest Path First)プロトコルがある。

このOSPFは管理者が設定する静的なリンクコスト値を用いた経路制御を行って
いるが、このリンクコスト値を利用状況に応じて動的に変えることで、より正
しい(効率の良い)経路制御を行うことを目指す。

OSPFに対応したgatedをPC-UNIX上で動作させ、動的にコスト値を変更すること
で経路制御にどう影響するかを考察した。
話者 : (2) 多田 好克

メモ: (1) 川崎 修 (電通大)
「帯域利用状況に対応した経路制御に関する考察」
  • Q.しょっちゅうダウンしている場合には動的と言えるのか

    A.ネットワークがしょっちゅうダウンしている場合にも、それに応じ て動作することになります。不安定な状態がずっと続くのですが、動 的と言えることになります。
  • Q.ルータが混雑した場合には x は変化するのでは

    A.ルータ処理時間 x はトラフィック量と性能に依存した値になります。 x の変動がかなり大きいルータ(非常に利用率の高いルータ)へ計測パ ケットを送信すると、大きいサイズのパケットより小さいサイズのRTT が長くなる現象があり、計測精度が落ちることがあります。
  • Q.ルータが一つのネットワークに多数接続されていても問題はないのか

    A.現在のOSPFの仕様では宛先ルータごとにコスト値を決めることはで きないため、セグメント内のどれかのルータを宛先ルータと考え、計 測パケットの宛先にしています。実際はリピータやLANスイッチ性能に よって、宛先ルータごとにネットワーク性能は異なるが、このシステ ムではブロードキャストの届く範囲は同じ性能として捉えています。
  • Q.パラメータが多すぎて何が効いているのかわからないのでシミュレー ションをして効果を振らしてみてはどうか

    A.まだ評価項目が少なく、パラメータの効果を十分評価できていませ ん。シミュレーションは引き続き行います。
  • Q.ネットワーク負荷の仮定が adhoc すぎる．観測したり，何らかのモ デルに基づくことをしては．

    A.確に実際のネットワークのトラフィック挙動から負荷データを作成 する方がより説得力があると考えます。しかし、ランダムを仮定して ポアソン分布などのモデルを使用した解析を行っても、計測結果にそ れほど変化はないと感じています。
